Navbar

Options that define the top navigation bar for a ?meta:project-type For example:

---
?meta:project-type:
  navbar:
    search: true
---

Nav Items

Nav items appear in the left or right key of navbar definitions. For example:

---
?meta:project-type:
  navbar:
    right:
      - icon: github
        href: https://github.com/
---

Sidebar

Options that define the side navigation area for a ?meta:project-type. For example:

---
?meta:project-type:
  sidebar:
    search: true
---

Sidebar Tools

Action buttons that appear on the sidebar. For example:

---
?meta:project-type:
  sidebar:
    tools:
      - icon: github
        href: https://github.com/
---

Footer

?meta:project-type-upper page footer definition. For example:

---
?meta:project-type:
  page-footer:
    center: 
      - text: "About"
        href: about.qmd
      - text: "License"
        href: license.qmd
      - text: "Trademark"
        href: trademark.qmd
---

Search

Search options are specified under the search key of ?meta:project-type. For example:

?meta:project-type:
  search:
    location: navbar
    type: overlay
---

Algolia Options

You can use an Algolia index as the back end of ?meta:project-type search. Specify Algolia options using the algolia sub-key of search, for example:

---
?meta:project-type:
  search:
    algolia:
      index-name: <my-index-name>
      application-id: <my-application-id>
      search-only-api-key: <my-search-only-api-key>
---

Social

Social metadata is provided as a subkey of ?meta:project-type options. You can specify true to generate social metadata using a set of default option, or specify one or more Twitter or Open Graph specific options as enumerated below. For example:

---
?meta:project-type:
  open-graph: true
  twitter-card: 
    site: "@sitehandle"
---

Comments

You can add commenting to your ?meta:project-type using either Hypothesis or Utterances.

Hypothesis

Enable and configure Hypothesis commenting via the comments key of ?meta:project-type. For example:

---
?meta:project-type:
  comments: 
    hypothesis:
      theme: clean
      openSidebar: false
---

For additional documentation on the Hypothesis options enumerated above, see the Hypothesis Publisher Config documentation.

Utterances

Enable and configure Utterances commenting via the comments key of ?meta:project-type. For example:

---
?meta:project-type:
  comments: 
    utterances:
      repo: quarto-dev/quarto-web
---
